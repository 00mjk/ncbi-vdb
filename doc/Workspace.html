Abstract

Workspace will implement a user-space file-system to provide transparent read-only access to remote or cached objects, in addition to a local read/write workspace volume.

Users enter a workspace by launching this tool, and by so doing gain access to NCBI remote resources as if locally mounted in their file system.

Entering a protected workspace, i.e. a dbGaP project, will create a mount where all remote and cached dbGaP objects will appear as fully decrypted objects, allowing them to be accessed by standard tools. In addition, their workspace volume will be encrypted.

After exiting a workspace, the user-space mounts will disappear, leaving only the actual OS file-system. All files within protected workspaces will remain encrypted, including the read/write workspace-volume.

Purpose

This utility addresses the need to provide unencumbered access to protected data while keeping the data reasonably well protected. It does not provide iron-clad protection, particularly in that it does not prevent the user from violating data confidentiality; but it does allow the user to preserve it.

The principal technique employed is the use of VDB on-the-fly decryption in response to requests for data. The tool runs as a process in Linux, MacOS or Windows, and acts as a bridge between the host kernel and VDB libraries. Once situated there, it can also provide additional services.

By utilizing networking connections with NCBI, the tool can also provide a mount of objects that are remotely stored on NCBI's servers. These include public as well as authorized-access objects. Working with the VDB KeyRing service, access authorization can be utilized to this end.

To assist users in working with confidential data while not compromising its confidentiality, we provide a means of creating a workspace-volume. This is nothing more than a normal file within the host file-system that is mounted under the workspace such that all writes to the file are encrypted and all reads from it are decrypted. The file itself will contain a portable VDB filesystem, the contents of which will be visible only after entering a workspace.

Going forward, we intend to provide custom mounts that represent the result of queries to NCBI's resources as tangible objects.

Similar in concept but separate in implementation, we may also provide the ability to view SRA objects as fastq as well as SAM. These present some level of difficulty that may outweigh their benefit, so they are only speculative.
